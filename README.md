# ERVU Analysis

Технический анализ архитектуры ЕРВУ (Единый реестр воинского учёта) — федеральной информационной системы РФ для централизованного учёта военнообязанных граждан.

## Содержание

Анализ включает:

- **Архитектура системы** — сетевая топология, ~40 микросервисов, технологический стек (Java, Kafka, MinIO, Camunda)
- **Модель данных** — структура записей о гражданах, документы, расширенные данные (семья, работа, образование, судимости)
- **Процесс повесток** — статусная модель, автоматическое вручение через 7 дней
- **Ограничительные меры** — запрет выезда, регистрации ТС/недвижимости, получения кредитов (автоматически через 20 дней после неявки)
- **Личные кабинеты** — ЛК ФЛ, ЛК ЮЛ, Реестр повесток
- **Интеграции** — СМЭВ, ФНС, МВД, СФР, ФСБ, ГИБДД, Росреестр, ЕСИА, ЕПГУ
- **Журналирование** — полный аудит всех действий и изменений
